스레드

struct thread

Pintos에서 스레드를 표현하는 핵심 자료구조는 struct thread입니다. 이 구조체는 threads/thread.h에 선언되어 있습니다.

struct thread;

하나의 스레드 또는 사용자 프로세스를 나타냅니다. 프로젝트를 진행하면서 필요한 필드를 struct thread에 직접 추가하게 됩니다. 필요하다면 기존 멤버의 정의를 바꾸거나 삭제해도 됩니다.

각 struct thread는 자기 전용 메모리 페이지의 시작 부분을 차지합니다. 그 페이지의 나머지 공간은 해당 스레드의 커널 스택으로 쓰이며, 스택은 페이지 끝에서 낮은 주소 방향으로 자랍니다.

                      4 kB +---------------------------------+
                           |         kernel stack            |
                           |               |                 |
                           |               |                 |
                           |               V                 |
                           |        grows downward           |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
                           |                                 |
    sizeof (struct thread) +---------------------------------+
                           |             magic               |
                           |          intr_frame             |
                           |               :                 |
                           |               :                 |
                           |             status              |
                           |              tid                |
                      0 kB +---------------------------------+

이 배치는 두 가지 제약을 만듭니다. 첫째, struct thread를 너무 크게 만들면 안 됩니다. 구조체가 커지면 같은 페이지 안에 있는 커널 스택 공간이 줄어듭니다. 기본 struct thread는 몇 바이트 수준이며, 보통 1 kB보다 충분히 작게 유지하는 것이 좋습니다.

둘째, 커널 스택도 너무 커지면 안 됩니다. 스택이 넘치면 같은 페이지 아래쪽에 있는 스레드 상태를 덮어써서 망가뜨립니다. 따라서 커널 함수 안에서 큰 구조체나 배열을 non-static local variable로 잡지 마세요. 그런 경우에는 malloc()이나 palloc_get_page()로 동적 할당을 사용해야 합니다. 자세한 내용은 메모리 할당을 참고하세요.

tid_t tid;

스레드 식별자(thread identifier), 즉 tid입니다. 모든 스레드의 tid는 커널이 살아 있는 전체 시간 동안 유일해야 합니다. 기본 코드에서 tid_t는 int의 typedef이고, 초기 프로세스가 1을 받으며 새 스레드는 숫자가 하나씩 증가한 tid를 받습니다. 원한다면 타입이나 번호 부여 방식은 바꿀 수 있습니다.

enum thread_status status;

스레드의 현재 상태입니다. 가능한 값은 다음과 같습니다.

  • THREAD_RUNNING

    스레드가 실행 중입니다. 어떤 순간에도 실행 중인 스레드는 정확히 하나입니다. thread_current()는 이 실행 중인 스레드를 반환합니다.

  • THREAD_READY

    실행할 준비는 되었지만 지금 실행 중은 아닌 상태입니다. 다음에 스케줄러가 호출되면 이 스레드가 선택될 수 있습니다. ready 상태의 스레드들은 ready_list라는 doubly linked list에 들어 있습니다.

  • THREAD_BLOCKED

    스레드가 어떤 사건을 기다리는 상태입니다. 예를 들어 lock이 풀리기를 기다리거나, 어떤 interrupt가 발생하기를 기다릴 수 있습니다. 이 스레드는 thread_unblock() 호출로 THREAD_READY 상태가 되기 전까지 다시 스케줄되지 않습니다. 보통은 Pintos의 동기화 primitive가 스레드의 block/unblock을 자동으로 처리하도록 사용하는 편이 가장 편합니다. 자세한 내용은 동기화를 참고하세요. blocked 스레드가 정확히 무엇을 기다리는지 상태값만 보고 미리 알 수 있는 일반적인 방법은 없지만, backtrace가 단서가 될 수 있습니다. Backtraces를 참고하세요.

  • THREAD_DYING

    스레드가 종료 중입니다. 다음 스레드로 context switch한 뒤 스케줄러가 이 스레드를 정리합니다.

char name[16];

스레드 이름 문자열입니다. 이름이 길면 앞부분 일부만 저장됩니다.

struct intr_frame tf;

context switching에 필요한 정보를 저장합니다. 레지스터 값과 스택 포인터가 여기에 포함됩니다.

int priority;

스레드 우선순위입니다. 범위는 PRI_MIN(0)부터 PRI_MAX(63)까지입니다. 숫자가 낮을수록 우선순위가 낮으므로 0이 가장 낮고 63이 가장 높습니다. 제공되는 Pintos는 처음에는 스레드 우선순위를 무시하지만, 프로젝트 1에서 우선순위 스케줄링을 구현하게 됩니다. 우선순위 스케줄링을 참고하세요.

struct list_elem elem;

스레드를 doubly linked list에 넣기 위한 "list element"입니다. 이 필드는 실행 준비가 된 스레드 목록인 ready_list에 들어갈 때도 쓰이고, sema_down()에서 semaphore를 기다리는 스레드 목록에 들어갈 때도 쓰입니다. semaphore를 기다리는 스레드는 ready 상태가 아니고, ready 상태인 스레드는 semaphore 대기 목록에 있지 않으므로 하나의 필드를 두 용도로 쓸 수 있습니다.

uint64_t *pml4;

프로젝트 2 이후에만 존재하는 필드입니다. 페이지 테이블을 참고하세요.

unsigned magic

항상 THREAD_MAGIC으로 설정됩니다. THREAD_MAGIC은 threads/thread.c에 정의된 임의의 값이며, stack overflow를 감지하는 데 쓰입니다. thread_current()는 실행 중인 스레드의 struct thread에서 magic 멤버가 THREAD_MAGIC인지 검사합니다. 스택이 넘치면 이 값이 바뀌기 쉬워서 assertion이 터집니다. 이 검사가 최대한 잘 동작하게 하려면 struct thread에 새 멤버를 추가하더라도 magic은 맨 끝에 두세요.

스레드 함수

threads/thread.c는 스레드를 지원하기 위한 여러 public function을 구현합니다. 자주 보게 될 함수들은 다음과 같습니다.

void thread_init (void);

main()이 스레드 시스템을 초기화하기 위해 호출합니다. 주된 목적은 Pintos의 initial thread를 위한 struct thread를 만드는 것입니다. Pintos loader가 initial thread의 스택을 다른 Pintos 스레드와 같은 방식으로 페이지 맨 위에 놓기 때문에 이것이 가능합니다.

thread_init()이 실행되기 전에는 실행 중인 스레드의 magic 값이 올바르지 않으므로 thread_current()가 실패합니다. lock_acquire() 같은 많은 함수가 직접 또는 간접적으로 thread_current()를 호출하므로, thread_init()은 Pintos 초기화 초반에 호출됩니다.

void thread_start (void);

main()이 스케줄러를 시작하기 위해 호출합니다. 먼저 idle thread를 만듭니다. idle thread는 실행 가능한 다른 스레드가 없을 때 스케줄되는 스레드입니다. 그 다음 interrupt를 켭니다. interrupt를 켜면 timer interrupt에서 돌아올 때 intr_yield_on_return()을 통해 스케줄러가 동작할 수 있으므로, 결과적으로 스케줄러가 활성화됩니다.

void thread_tick (void);

timer interrupt가 매 tick마다 호출합니다. 스레드 통계를 관리하고, time slice가 끝나면 스케줄러가 실행되도록 만듭니다.

void thread_print_stats (void);

Pintos가 종료될 때 호출되어 스레드 통계를 출력합니다.

tid_t thread_create (const char *name, int priority, thread func *func, void *aux);

주어진 name과 priority를 가진 새 스레드를 만들고 시작한 뒤, 새 스레드의 tid를 반환합니다. 새 스레드는 func를 실행하며, aux는 그 함수의 단일 인자로 전달됩니다.

thread_create()는 새 스레드의 struct thread와 stack을 담을 페이지를 할당하고 멤버들을 초기화합니다. 그런 다음 새 스레드가 처음 실행될 수 있도록 가짜 stack frame들을 구성합니다. 새 스레드는 처음에는 blocked 상태로 초기화되고, 함수가 반환되기 직전에 unblock되어 스케줄될 수 있게 됩니다.

void thread_func (void *aux);

thread_create()에 넘기는 함수의 타입입니다. aux 인자는 그 함수의 인자로 그대로 전달됩니다.

void thread_block (void);

실행 중인 스레드를 running 상태에서 blocked 상태로 전환합니다. 이 스레드는 자신에게 thread_unblock()이 호출되기 전까지 다시 실행되지 않습니다. 따라서 unblock될 경로가 반드시 준비되어 있어야 합니다. thread_block()은 매우 낮은 수준의 함수이므로, 일반적으로는 직접 쓰기보다 동기화 primitive를 사용하는 편이 좋습니다.

void thread_unblock (struct thread *thread);

blocked 상태여야 하는 thread를 ready 상태로 바꾸어 다시 실행될 수 있게 합니다. 예를 들어 스레드가 기다리던 lock을 사용할 수 있게 되는 등, 기다리던 사건이 발생했을 때 호출됩니다.

struct thread *thread_current (void);

현재 실행 중인 스레드를 반환합니다.

tid_t thread_tid (void);

현재 실행 중인 스레드의 tid를 반환합니다. thread_current ()->tid와 같습니다.

const char *thread_name (void);

현재 실행 중인 스레드의 이름을 반환합니다. thread_current ()->name과 같습니다.

void thread_exit (void) NO_RETURN;

현재 스레드를 종료합니다. 이 함수는 돌아오지 않습니다.

void thread_yield (void);

CPU를 스케줄러에게 양보합니다. 스케줄러는 실행할 새 스레드를 고릅니다. 선택된 "새" 스레드가 현재 스레드일 수도 있으므로, 이 함수를 호출했다고 해서 현재 스레드가 특정 시간 동안 반드시 실행을 멈춘다고 가정하면 안 됩니다.

int thread_get_priority (void);
void thread_set_priority (int new_priority);

스레드 우선순위를 읽고 설정하기 위한 stub입니다. 우선순위 스케줄링을 참고하세요.

int thread_get_nice (void);
void thread_set_nice (int new_nice);
int thread_get_recent_cpu (void);
int thread_get_load_avg (void);

advanced scheduler를 위한 stub들입니다.
