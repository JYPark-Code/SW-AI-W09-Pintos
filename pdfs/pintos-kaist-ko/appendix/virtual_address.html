가상 주소

64비트 가상 주소는 다음과 같은 구조를 가집니다.

63          48 47            39 38            30 29            21 20         12 11         0
+-------------+----------------+----------------+----------------+-------------+------------+
| Sign Extend |    Page-Map    | Page-Directory | Page-directory |  Page-Table |  Physical  |
|             | Level-4 Offset |    Pointer     |     Offset     |   Offset    |   Offset   |
+-------------+----------------+----------------+----------------+-------------+------------+
              |                |                |                |             |            |
              +------- 9 ------+------- 9 ------+------- 9 ------+----- 9 -----+---- 12 ----+
                                          Virtual Address

include/threads/vaddr.h와 include/threads/mmu.h 헤더는 가상 주소를 다루기 위한 다음 함수와 매크로를 정의합니다.

#define PGSHIFT { /* Omit details */ }
#define PGBITS { /* Omit details */ }

가상 주소의 offset 부분에 대한 bit index(0)와 bit 수(12)를 각각 나타냅니다.

#define PGMASK { /* Omit details */ }

page offset에 해당하는 bit는 1이고 나머지는 0인 bit mask입니다(0xfff).

#define PGSIZE { /* Omit details */ }

page 크기입니다. 바이트 단위로 4,096입니다.

#define pg_ofs(va) { /* Omit details */ }

가상 주소 va에서 page offset을 추출해 반환합니다.

#define pg_no(va) { /* Omit details */ }

가상 주소 va에서 page number를 추출해 반환합니다.

#define pg_round_down(va) { /* Omit details */ }

va가 속한 virtual page의 시작 주소를 반환합니다. 즉 va에서 page offset을 0으로 만든 값입니다.

#define pg_round_up(va) { /* Omit details */ }

va를 가장 가까운 page boundary로 올림한 값을 반환합니다.

Pintos의 가상 메모리는 user virtual memory와 kernel virtual memory 두 영역으로 나뉩니다. 가상 메모리 배치를 참고하세요.

두 영역의 경계는 KERN_BASE입니다.

#define KERN_BASE { /* Omit details */ }

kernel virtual memory의 base address입니다. 기본값은 0x8004000000입니다. user virtual memory는 가상 주소 0부터 KERN_BASE까지이고, kernel virtual memory는 나머지 가상 주소 공간을 차지합니다.

#define is_user_vaddr(vaddr) { /* Omit details */ }
#define is_kernel_vaddr(vaddr) { /* Omit details */ }

va가 각각 user 또는 kernel virtual address이면 true를, 그렇지 않으면 false를 반환합니다.

x86-64는 physical address가 주어졌을 때 그 메모리에 직접 접근하는 방법을 제공하지 않습니다. OS 커널에서는 이 능력이 자주 필요하므로, Pintos는 kernel virtual memory를 physical memory에 일대일로 mapping해서 해결합니다. 즉 가상 주소 KERN_BASE는 physical address 0에 접근하고, 가상 주소 KERN_BASE + 0x1234는 physical address 0x1234에 접근하는 식으로 머신의 physical memory 크기까지 이어집니다. 따라서 physical address에 KERN_BASE를 더하면 그 주소에 접근하는 kernel virtual address를 얻을 수 있습니다. 반대로 kernel virtual address에서 KERN_BASE를 빼면 대응하는 physical address를 얻습니다.

include/threads/vaddr.h 헤더는 이 변환을 수행하는 두 함수를 제공합니다.

#define ptov(paddr) { /* Omit details */ }

physical address pa에 대응하는 kernel virtual address를 반환합니다. pa는 0과 physical memory의 바이트 수 사이에 있어야 합니다.

#define vtop(vaddr) { /* Omit details */ }

va에 대응하는 physical address를 반환합니다. va는 kernel virtual address여야 합니다.

include/threads/mmu.h 헤더는 page table에 대한 연산을 제공합니다.

#define is_user_pte(pte) { /* Omit details */ }
#define is_kern_pte(pte) { /* Omit details */ }

page table entry(PTE)가 각각 user 소유인지 kernel 소유인지 확인합니다.

#define is_writable(pte) { /* Omit details */ }

page table entry(PTE)가 가리키는 가상 주소가 writable인지 확인합니다.

typedef bool pte_for_each_func (uint64_t *pte, void *va, void *aux);
bool pml4_for_each (uint64_t *pml4, pte_for_each_func *func, void *aux);

PML4 아래의 각 valid entry에 대해 보조 값 AUX와 함께 FUNC를 적용합니다. VA는 해당 entry의 가상 주소를 나타냅니다. pte_for_each_func가 false를 반환하면 순회를 멈추고 false를 반환합니다.

아래는 pml4_for_each에 넘길 수 있는 func의 예입니다.

static bool
stat_page (uint64_t *pte, void *va,  void *aux) {
        if (is_user_vaddr (va))
                printf ("user page: %llx\n", va);
        if (is_writable (va))
                printf ("writable page: %llx\n", va);
        return true;
}
