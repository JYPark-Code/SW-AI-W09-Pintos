페이지 테이블

threads/mmu.c의 코드는 x86_64 하드웨어 page table에 대한 추상 인터페이스입니다. Pintos에서 사용할 page table은 Intel 프로세서 문서에서 Page-Map-Level-4의 약자인 pml4라고 부릅니다. 이 table이 4단계 구조이기 때문입니다. page table 인터페이스는 내부 구조에 접근하기 편하도록 uint64_t *를 사용해 page table을 표현합니다. 아래 절들은 page table 인터페이스와 내부 동작을 설명합니다.

생성, 파괴, 활성화

다음 함수들은 page table을 만들고, 파괴하고, 활성화합니다. 기본 Pintos 코드는 필요한 곳에서 이미 이 함수들을 호출하므로 직접 호출할 필요는 보통 없습니다.

uint64_t * pml4_create (void);

새 page table을 만들고 반환합니다. 새 page table에는 Pintos의 일반적인 kernel virtual page mapping은 들어 있지만 user virtual mapping은 없습니다. 메모리를 얻을 수 없으면 null pointer를 반환합니다.

void pml4_destroy (uint64_t *pml4);

pml4가 보유한 모든 자원을 해제합니다. 여기에는 page table 자체와 그것이 mapping하는 frame들이 포함됩니다. page table의 모든 level에서 자원을 해제하기 위해 pdpe_destroy, pgdir_destory, pt_destroy를 재귀적으로 호출합니다.

void pml4_activate (uint64 t *pml4)

pml4를 활성화합니다. active page table은 CPU가 메모리 참조를 변환할 때 사용하는 page table입니다.

검사와 갱신

다음 함수들은 page table이 담고 있는 page-to-frame mapping을 검사하거나 갱신합니다. 실행 중인 프로세스의 active page table과 중단된 프로세스의 inactive page table 모두에서 동작하며, 필요하면 TLB를 flush합니다.

bool pml4_set_page (uint64_t *pml4, void *upage, void *kpage, bool rw);

user page upage에서 kernel virtual address kpage로 식별되는 frame으로의 mapping을 pml4에 추가합니다. rw가 true이면 page는 read/write로 mapping되고, 그렇지 않으면 read-only로 mapping됩니다. user page upage는 pml4에 이미 mapping되어 있으면 안 됩니다. kernel page kpage는 palloc_get_page(PAL_USER)로 user pool에서 얻은 kernel virtual address여야 합니다. 성공하면 true, 실패하면 false를 반환합니다. 실패는 page table에 필요한 추가 메모리를 얻을 수 없을 때 발생합니다.

void * pml4_get_page (uint64_t *pml4, const void *uaddr);

pml4에서 uaddr에 mapping된 frame을 찾습니다. uaddr가 mapping되어 있으면 그 frame의 kernel virtual address를 반환하고, mapping되어 있지 않으면 null pointer를 반환합니다.

void pml4_clear_page (uint64_t *pml4, void *upage);

pml4에서 page를 "not present"로 표시합니다. 이후 그 page에 접근하면 fault가 발생합니다. page table 안의 다른 bit들은 보존되므로 accessed bit와 dirty bit를 나중에 확인할 수 있습니다. page가 mapping되어 있지 않으면 이 함수는 아무 효과가 없습니다.

Accessed Bit와 Dirty Bit

x86_64 하드웨어는 각 page의 page table entry(PTE)에 있는 두 bit를 통해 page replacement algorithm 구현을 어느 정도 지원합니다. 어떤 page를 읽거나 쓰면 CPU는 그 page의 PTE에서 accessed bit를 1로 설정합니다. 어떤 page에 쓰면 dirty bit를 1로 설정합니다. CPU는 이 bit들을 절대 0으로 되돌리지 않지만, OS는 그렇게 할 수 있습니다.

이 bit들을 올바르게 해석하려면 alias를 이해해야 합니다. alias란 둘 이상의 page가 같은 frame을 참조하는 경우입니다. alias된 frame에 접근하면 accessed bit와 dirty bit는 접근에 사용된 page의 page table entry 하나에서만 갱신됩니다. 다른 alias의 accessed bit와 dirty bit는 갱신되지 않습니다.

bool pml4_is_dirty (uint64_t *pml4, const void *vpage);
bool pml4_is_accessed (uint64_t *pml4, const void *vpage);

pml4에 vpage에 대한 page table entry가 있고 그 entry가 dirty 또는 accessed로 표시되어 있으면 true를 반환합니다. 그렇지 않으면 false를 반환합니다.

void pml4_set_dirty (uint64_t *pml4, const void *vpage, bool dirty);
void pml4_set_accessed (uint64_t *pml4, const void *vpage, bool accessed);

pml4에 해당 page에 대한 page table entry가 있으면, 그 dirty bit 또는 accessed bit를 주어진 값으로 설정합니다.
