파일 디스크립터 확장 (추가 과제)

Pintos가 Linux처럼 stdin, stdout 닫기와 dup2 시스템 콜을 지원하도록 만듭니다.

현재 Pintos 구현에서는 stdin과 stdout의 fd를 닫는 것이 금지되어 있습니다. 이 추가 점수 요구사항에서는 먼저 Linux와 같이 사용자가 stdin과 stdout을 닫을 수 있게 해야 합니다. 즉 stdin을 닫으면 프로세스는 더 이상 입력을 읽지 않아야 하고, stdout을 닫으면 프로세스는 아무것도 출력하지 않아야 합니다.

다음으로 dup2 시스템 콜을 구현합니다.

int dup2(int oldfd, int newfd);

dup2() 시스템 콜은 파일 디스크립터 oldfd의 복사본을 만들되, 그 복사본의 파일 디스크립터 번호를 newfd로 지정합니다. 성공하면 newfd를 반환합니다. newfd가 이미 열려 있었다면, 재사용되기 전에 조용히 닫힙니다.

다음 사항에 주의하세요.

  • oldfd가 유효한 파일 디스크립터가 아니면 호출은 실패하고 -1을 반환합니다. 이 경우 newfd는 닫히지 않습니다.

  • oldfd가 유효한 파일 디스크립터이고 newfd가 oldfd와 같은 값이면, dup2()는 아무 일도 하지 않고 newfd를 반환합니다.

이 시스템 콜이 성공적으로 반환된 뒤에는 기존 파일 디스크립터와 새 파일 디스크립터를 서로 바꿔 사용할 수 있습니다. 둘은 서로 다른 파일 디스크립터이지만 같은 open file description을 참조하므로 파일 offset과 status flag를 공유합니다. 예를 들어 한 디스크립터에서 seek를 사용해 파일 offset을 바꾸면, 다른 디스크립터의 offset도 함께 바뀝니다.

dup된 파일 디스크립터는 fork 이후에도 그 의미를 유지해야 합니다.

추가 점수를 받으려면 추가 과제의 모든 테스트 케이스를 통과해야 합니다. 전부 통과하지 못하면 점수를 받지 못합니다.
