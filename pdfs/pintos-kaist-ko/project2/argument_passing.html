인자 전달

process_exec()에서 사용자 프로그램의 인자를 설정합니다.

x86-64 호출 규약

이 절은 Unix의 64비트 x86-64 구현에서 일반 함수 호출에 사용하는 규약 중 중요한 부분을 요약합니다. 간결하게 설명하기 위해 일부 세부 사항은 생략합니다. 더 자세한 내용은 System V AMD64 ABI를 참고할 수 있습니다.

호출 규약은 다음과 같이 동작합니다.

  1. 사용자 수준 애플리케이션은 정수 인자를 전달할 때 순서대로 %rdi, %rsi, %rdx, %rcx, %r8, %r9 레지스터를 사용합니다.
  2. 호출자는 다음에 실행할 명령어의 주소, 즉 반환 주소를 스택에 push하고 callee의 첫 명령어로 점프합니다. x86-64의 CALL 명령 하나가 이 두 동작을 수행합니다.
  3. callee가 실행됩니다.
  4. callee에 반환 값이 있으면 그 값을 RAX 레지스터에 저장합니다.
  5. callee는 스택에서 반환 주소를 pop하고, x86-64 RET 명령으로 그 주소가 가리키는 위치로 점프해서 반환합니다.

세 개의 int 인자를 받는 함수 f()를 생각해 봅시다. f()가 f(1, 2, 3)으로 호출되었다고 할 때, 위 3단계가 시작되는 시점에 callee가 보는 스택 프레임과 레지스터 상태는 아래 예시처럼 됩니다. 초기 스택 주소는 임의의 값입니다.

                             +----------------+
stack pointer --> 0x4747fe70 | return address |
                             +----------------+
RDI: 0x0000000000000001 | RSI: 0x0000000000000002 | RDX: 0x0000000000000003

프로그램 시작 세부 사항

사용자 프로그램용 Pintos C 라이브러리는 lib/user/entry.c의 _start()를 사용자 프로그램의 진입점으로 지정합니다. 이 함수는 main()을 감싸는 래퍼이며, main()이 반환하면 exit()을 호출합니다.

void
_start (int argc, char *argv[]) {
    exit (main (argc, argv));
}

커널은 사용자 프로그램 실행을 시작하기 전에 초기 함수의 인자들을 레지스터에 넣어야 합니다. 인자는 일반 호출 규약과 같은 방식으로 전달됩니다.

예제 명령 /bin/ls -l foo bar의 인자를 어떻게 처리할지 생각해 봅시다.

  1. 명령을 단어로 나눕니다: /bin/ls, -l, foo, bar.

  2. 단어들을 스택 맨 위에 놓습니다. 이 단어들은 포인터를 통해 참조될 것이므로, 문자열 자체를 놓는 순서는 중요하지 않습니다.

  3. 각 문자열의 주소와 null pointer sentinel을 오른쪽에서 왼쪽 순서로 스택에 push합니다. 이것들이 argv의 원소가 됩니다. null pointer sentinel은 C 표준이 요구하는 것처럼 argv[argc]가 null pointer가 되도록 보장합니다. 오른쪽에서 왼쪽으로 넣는 이유는 argv[0]이 가장 낮은 가상 주소에 놓이도록 하기 위해서입니다. 워드 정렬된 접근은 정렬되지 않은 접근보다 빠르므로, 성능을 위해 첫 push 전에 스택 포인터를 8의 배수로 내림 정렬합니다.

  4. %rsi가 argv, 즉 argv[0]의 주소를 가리키게 하고, %rdi를 argc로 설정합니다.

  5. 마지막으로 가짜 "return address"를 push합니다. 진입 함수는 실제로 반환하지 않지만, 그 스택 프레임은 다른 함수의 스택 프레임과 같은 구조를 가져야 합니다.

아래 표는 사용자 프로그램이 시작되기 직전의 스택 상태와 관련 레지스터를 보여 줍니다. 스택은 아래쪽으로 자란다는 점에 주의하세요.

이 예에서 스택 포인터는 0x4747ffb8로 초기화됩니다. 위에서 보듯이 코드는 include/threads/vaddr.h에 정의된 USER_STACK에서 스택을 시작해야 합니다.

<stdio.h>에 선언된 비표준 함수 hex_dump()는 인자 전달 코드를 디버깅할 때 유용할 수 있습니다.

인자 전달 구현

현재 process_exec()는 새 프로세스에 인자를 전달하는 기능을 지원하지 않습니다. 이 기능을 구현하려면 process_exec()를 확장해서, 인자를 단순히 프로그램 파일명 하나로 취급하지 말고 공백 기준으로 단어들을 나누게 해야 합니다. 첫 번째 단어는 프로그램 이름이고, 두 번째 단어는 첫 번째 인자, 그 다음 단어는 두 번째 인자입니다. 즉 process_exec("grep foo bar")는 grep을 실행하면서 foo와 bar라는 두 인자를 넘겨야 합니다.

명령줄 안에서 여러 개의 공백은 하나의 공백과 같습니다. 따라서 공백이 여러 개 들어간 명령도 원래 예제와 동일하게 처리되어야 합니다. 명령줄 인자 길이에는 합리적인 제한을 둘 수 있습니다. 예를 들어 인자들이 한 페이지(4 kB)에 들어갈 때까지만 허용할 수 있습니다. 이 제한과 별개로, pintos 유틸리티가 커널에 전달할 수 있는 명령줄 인자에는 128바이트 제한이 있습니다.

인자 문자열은 원하는 방식으로 파싱해도 됩니다. 막막하다면 include/lib/string.h에 프로토타입이 있고 lib/string.c에 자세한 주석과 함께 구현된 strtok_r()를 살펴보세요. 프롬프트에서 man strtok_r를 실행해 man page를 봐도 더 많은 정보를 얻을 수 있습니다.
