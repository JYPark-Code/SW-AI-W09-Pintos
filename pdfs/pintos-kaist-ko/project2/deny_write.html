실행 파일 쓰기 거부

실행 파일로 사용 중인 파일에는 쓰기를 거부합니다.

실행 파일로 사용 중인 파일에 쓰지 못하도록 코드를 추가하세요. 많은 OS가 이렇게 합니다. 디스크에서 변경되고 있는 코드를 프로세스가 실행하려고 하면 예측하기 어려운 결과가 생길 수 있기 때문입니다. 이 처리는 프로젝트 3에서 가상 메모리가 구현된 뒤 특히 중요하지만, 지금 적용해도 나쁠 것은 없습니다.

열린 파일에 대한 쓰기를 막으려면 file_deny_write()를 사용할 수 있습니다. 같은 파일에 file_allow_write()를 호출하면 다시 쓰기를 허용합니다. 단, 다른 opener가 여전히 쓰기를 거부하고 있다면 허용되지 않습니다. 파일을 닫아도 쓰기가 다시 허용됩니다. 따라서 어떤 프로세스의 실행 파일에 쓰기를 거부하려면, 그 프로세스가 실행 중인 동안 해당 파일을 계속 열어 두어야 합니다.

이전: 프로세스 종료 메시지 다음: 파일 디스크립터 확장 (추가 과제)
