Swap In/Out

memory swapping은 물리 메모리 사용을 극대화하기 위한 메모리 회수 기법입니다. 주 메모리의 frame이 모두 할당되면 시스템은 사용자 프로그램의 추가 메모리 할당 요청을 더 이상 처리할 수 없습니다. 한 가지 해결책은 현재 사용되지 않는 memory frame을 디스크로 swap out하는 것입니다. 이렇게 하면 일부 메모리 자원이 비워져 다른 애플리케이션이 사용할 수 있습니다.

swapping은 운영체제가 수행합니다. 시스템이 메모리가 부족한 상태에서 메모리 할당 요청을 받으면, swap disk로 내보낼 page를 선택합니다. 그런 다음 memory frame의 정확한 상태를 디스크에 복사합니다. 프로세스가 swap out된 page에 접근하려고 하면, OS는 그 page의 정확한 내용을 다시 메모리로 가져와 복구합니다.

eviction 대상으로 선택되는 page는 anonymous page일 수도 있고 file-backed page일 수도 있습니다. 이 절에서는 두 경우를 각각 처리합니다.

모든 swapping 연산은 명시적으로 직접 호출되지 않고 함수 포인터를 통해 호출됩니다. 이 함수 포인터들은 struct page_operations의 멤버이며, 각 page initializer에서 그 page의 operations로 등록됩니다.

Anonymous Page

vm/anon.c의 vm_anon_init과 anon_initializer를 수정하세요. anonymous page에는 backing storage가 없습니다. anonymous page의 swapping을 지원하기 위해 swap disk라는 임시 backing storage를 제공합니다. anonymous page의 swap을 구현할 때 이 swap disk를 사용합니다.

void vm_anon_init (void);

이 함수에서는 swap disk를 설정해야 합니다. 또한 swap disk에서 비어 있는 영역과 사용 중인 영역을 관리할 자료구조가 필요합니다. swap 영역도 PGSIZE(4096바이트) 단위로 관리됩니다.

bool anon_initializer (struct page *page, enum vm_type type, void *kva);

anonymous page의 initializer입니다. swapping을 지원하려면 anon_page에 정보를 추가해야 합니다.

이제 anonymous page swapping을 지원하기 위해 vm/anon.c에 anon_swap_in과 anon_swap_out을 구현하세요. 어떤 page가 swap in되려면 먼저 swap out되어 있어야 하므로, anon_swap_in보다 anon_swap_out을 먼저 구현하는 편이 좋을 수 있습니다. 데이터 내용을 swap disk로 옮기고, 안전하게 다시 메모리로 가져와야 합니다.

static bool anon_swap_in (struct page *page, void *kva);

swap disk에서 데이터 내용을 읽어 메모리로 가져와 anonymous page를 swap in합니다. 데이터가 swap disk의 어디에 있는지는 page가 swap out될 때 page 구조체 안에 저장되어 있어야 합니다. swap table도 갱신해야 한다는 점을 기억하세요. Swap Table 관리를 참고하세요.

static bool anon_swap_out (struct page *page);

메모리의 내용을 디스크로 복사해서 anonymous page를 swap out합니다. 먼저 swap table을 사용해 디스크에서 빈 swap slot을 찾고, 그 slot에 page 데이터 한 페이지를 복사합니다. 데이터의 위치는 page 구조체에 저장해야 합니다. 디스크에 더 이상 빈 slot이 없으면 커널을 panic시킬 수 있습니다.

File-Mapped Page

file-backed page의 내용은 파일에서 오므로, mmap된 파일 자체를 backing store로 사용해야 합니다. 즉 file-backed page를 evict하면 그 page는 자신이 mapping된 파일에 다시 기록됩니다. vm/file.c에 file_backed_swap_in과 file_backed_swap_out을 구현하세요. 설계에 따라 file_backed_init과 file_initializer를 수정할 수도 있습니다.

static bool file_backed_swap_in (struct page *page, void *kva);

파일에서 내용을 읽어 kva의 page로 swap in합니다. 파일 시스템과 동기화해야 합니다.

static bool file_backed_swap_out (struct page *page);

page의 내용을 파일에 다시 써서 swap out합니다. 먼저 page가 dirty인지 확인하는 것이 좋습니다. dirty가 아니라면 파일의 내용을 수정할 필요가 없습니다. page를 swap out한 뒤에는 그 page의 dirty bit를 끄는 것을 잊지 마세요.
