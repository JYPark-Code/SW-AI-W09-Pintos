FAQ

프로젝트 3을 구현하려면 동작하는 프로젝트 2가 필요한가요?

네.

page fault를 처리한 뒤 프로세스는 어떻게 재개하나요?

page_fault()에서 반환하면 현재 사용자 프로세스가 재개됩니다. 그러면 instruction pointer가 가리키던 명령어를 다시 시도합니다.

사용자 프로세스가 때때로 stack pointer 위쪽 주소에서 fault를 내는 이유도 궁금할 수 있습니다. stack growth 테스트에서는 사용자 프로그램이 현재 stack pointer보다 위에 있는 주소에서 fault를 내는 것을 볼 수 있습니다.

가상 메모리 시스템이 data segment 증가를 지원해야 하나요?

아니요. data segment의 크기는 링커가 결정합니다. Pintos에는 아직 동적 할당이 없습니다. memory-mapped file을 사용해 사용자 수준에서 이를 "흉내"낼 수는 있습니다. 잘 설계된 시스템이라면 data segment 증가 지원은 복잡도를 크게 늘리지 않을 것입니다.

page frame 할당에 왜 PAL_USER를 사용해야 하나요?

palloc_get_page()에 PAL_USER를 전달하면 기본 kernel pool이 아니라 user pool에서 메모리를 할당합니다. user pool의 page가 부족해지면 사용자 프로그램이 paging하게 될 뿐이지만, kernel pool의 page가 부족해지면 많은 커널 함수가 메모리를 얻어야 하므로 여러 실패가 발생합니다. 원한다면 palloc_get_page() 위에 다른 allocator를 얹을 수 있지만, 그 아래 기본 메커니즘은 이것이어야 합니다.

또한 -ul 커널 command-line option을 사용해 user pool 크기를 제한할 수 있습니다. 이렇게 하면 여러 사용자 메모리 크기에서 VM 구현을 쉽게 테스트할 수 있습니다.
