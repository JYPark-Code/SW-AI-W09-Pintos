학습 가이드

문서 첫 페이지

읽는 순서

  1. Introduction에서 환경, 채점, 윤리 규칙을 먼저 확인합니다.
  2. Project 1은 스레드, 인터럽트, 동기화 원리를 코드와 연결해서 읽습니다.
  3. Project 2는 사용자 프로그램이 커널로 진입하는 경로와 시스템 콜 계약을 중심으로 봅니다.
  4. Project 3은 페이지, 프레임, 보조 페이지 테이블, lazy loading, swap의 관계를 먼저 그려보고 구현합니다.

용어 번역 기준

원문번역읽는 포인트
thread스레드커널 안에서 스케줄링되는 실행 단위입니다.
semaphore / lock / condition variable세마포어 / 락 / 조건 변수동기화 primitive입니다. 구현보다 사용 규칙을 먼저 잡는 것이 중요합니다.
priority donation우선순위 기부락을 가진 낮은 우선순위 스레드에게 높은 우선순위를 임시로 전달합니다.
system call시스템 콜사용자 프로그램이 커널 기능을 요청하는 ABI 경계입니다.
page / frame페이지 / 프레임가상 메모리 단위와 물리 메모리 단위를 구분해서 읽어야 합니다.
supplemental page table보조 페이지 테이블하드웨어 페이지 테이블에 없는 lazy loading 정보를 커널이 보관하는 구조입니다.
eviction / swap축출 / 스왑물리 프레임이 부족할 때 어떤 페이지를 내보낼지 결정하는 흐름입니다.
inode / directory entryinode / 디렉터리 엔트리파일의 메타데이터와 이름 매핑을 분리해서 봅니다.

구현 전 체크

  • 테스트 하나를 고르고 입력, 기대 출력, 실패 시점부터 역추적합니다.
  • 공유 자료구조를 수정하는 함수마다 인터럽트 상태나 락 보유 조건을 적어 둡니다.
  • 프로젝트 2 이후에는 사용자 포인터 검증과 자원 해제 경로를 먼저 설계합니다.
  • VM에서는 page fault가 정상 lazy loading인지, 프로세스 종료 대상인지 먼저 분기합니다.
